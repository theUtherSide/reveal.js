Demandbase Tag

Tag Admin + JS API

Lunch n' Learn by Matthew Downs

What is a Demandbase Tag?

  • JavaScript files hosted in S3/CloudFront
  • Runs on (nearly) every page of every customer's website

Why Demandbase Tag?

  • v2012: Customer gets API docs and self-implements
  • v2013: Boilerplate JS given to customers for deployment
  • vNow:
    • Maintainability: Central place to manage distributed JS code
    • Scalability: small, static files delivered super fast over CDN
    • Deployability: Customer deploys once, Demandbase maintains remotely
    • Abstraction: shared core modules with custom configuration instead of ad-hoc implementations.

Why JavaScript on Customer Websites?

  • Reporting
    Creates/sends Page View data by calling the IP API
  • Advertising
    Load pixels from IPOW (view, retargeting, and conversion)
  • Integrating
    Company Profile into 3rd party systems

Size and Scale

  • > 625 tags (Accounts)
  • ~ 500M file requests / month
  • > 6000 GB transferred
  • Median File Size: ~14kb
  • Most-Active Account:
    Dropbox > 187M requests / month
  • Forms Module: > 12M requests / month

How are Tags Built?

2 repos involved:

  • js-api - JavaScript modules that are available to use in Tags.
  • tag - Admin application for creating and configuring Tags on each Account.

JS API - Integrations a.k.a. "Connectors"

  • JavaScript modules that can be added to a customer's Tag file.
  • Ready-made, configurable integrations into 3rd party applications

Available Connectors/Integrations

  • Google Analytics
  • WebTrends Analytics
  • Adobe Audience Manager
  • Forms:
    • Marketing Automation Systems: (any/all)
      • Eloqua
      • Marketo
      • Pardot
      • ExactTarget
      • NeoLane
    • works with any web form
    • includes Company Autocomplete Widget
  • Content:
    • Monetate
    • JS-based Segments/Content modules

Other Modules/Functionality

  • Ads (includes Campaigns, Conversions and Events)
  • Extended Company Targeting
  • Job Function Targeting
  • Domain API Identification

Tag Admin

  1. Application
    • Used by Services and Support teams to create each customer's JS file.
    • Set available modules and define custom configuration.
  2. API
    • Microservice that performs actions on Tags
    • Create new Accounts, Update modules/configurations, deploy to S3
